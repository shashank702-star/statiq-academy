Looker LookML Modeling

A reference guide on Looker's LookML data modeling language, defining views, explore joins, dimensions, and custom measures.

🏢 The Looker Modeling Layer

LookML separates SQL database logic from the visualization layer. Developers define schemas, joins, and measures in LookML files, and Looker generates optimized SQL queries dynamically on demand.

1. Defining a View (customer.view.lkml)

Views represent tables or derived tables in your database. Define fields as either dimensions (categories/variables) or measures (aggregates).

view: customer {
  sql_table_name: raw_warehouse.customer_api ;;

  # Dimension: Customer ID (Primary Key)
  dimension: customer_id {
    primary_key: yes
    type: string
    sql: ${TABLE}.id ;;
  }

  # Dimension: Customer Segment
  dimension: customer_segment {
    type: string
    sql: ${TABLE}.segment ;;
  }

  # Measure: Count of active customers
  measure: count {
    type: count
    drill_fields: [customer_id, customer_segment]
  }
}

2. Exploring and Joining (statiq.model.lkml)

Explores define the entry points for analysis and declare how different view files join together.

connection: "snowflake_dw"
include: "/views/*.view.lkml"

explore: orders {
  label: "Sales Ledger Analytics"

  # Join customer view file
  join: customer {
    type: left_outer
    relationship: many_to_one
    sql_on: ${orders.customer_id} = ${customer.customer_id} ;;
  }
}

3. LookML Custom Measures

Create dynamic aggregates that generate safe, grouped SQL formulas when selected by users.

view: orders {
  sql_table_name: raw_warehouse.orders ;;

  # Dimension: Raw order sale amount
  dimension: sales_usd {
    type: number
    sql: ${TABLE}.sales ;;
  }

  # Measure: Total revenue sum
  measure: total_revenue {
    type: sum
    sql: ${sales_usd} ;;
    value_format_name: usd_0
  }

  # Measure: Average order value
  measure: average_order_value {
    type: average
    sql: ${sales_usd} ;;
    value_format_name: usd
  }
}
