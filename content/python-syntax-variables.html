Python Syntax & Variables

Python is an interpreted, high-level, dynamically typed language. Its clean syntax relies on indentation rather than brackets or semicolons, making it readable and standard for data science, analytics, and software engineering.

1. Variables and Dynamic Typing

In Python, you do not declare a variable's data type explicitly. The type is inferred at runtime when a value is assigned. Variable names are case-sensitive and must start with a letter or underscore.

# Dynamic variable declarations
age = 28               # Inferred as Integer (int)
salary = 85400.75      # Inferred as Floating-point (float)
name = "Taylor"        # Inferred as String (str)
is_active = True       # Inferred as Boolean (bool)

# You can check types with the type() function
print(type(age))       # <class 'int'>
print(type(name))      # <class 'str'>

2. Basic Data Types & Casting

Python supports several primitive data types. Sometimes, you must explicitly convert one type to another (casting):

  • int(): Casts floats or compatible strings to integers (truncates decimals).
  • float(): Casts integers or compatible strings to floating-point numbers.
  • str(): Converts any data type (numbers, booleans) to a string representation.
# Casting examples
str_price = "199.99"
price = float(str_price)   # Cast string to float: 199.99

item_count = "15"
count = int(item_count)    # Cast string to integer: 15

# Type conversion error safety
try:
    invalid_cast = int("hello")
except ValueError as e:
    print(f"Casting failed: {e}")

3. String Formatting & Printing

The standard way to interpolate variables into printed text is using **f-strings** (formatted string literals). Prefix a string with f or F and write variables directly inside curly braces { }.

# f-string interpolation
first_name = "Jane"
last_name = "Doe"
score = 98.456

# Standard f-string
print(f"Student: {first_name} {last_name}")

# Padded decimals formatting
print(f"Final Score: {score:.2f}%") # Renders to: Final Score: 98.46%
