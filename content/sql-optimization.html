Modern SQL Optimization Cheatsheet

A reference guide for high-speed database transformations, cohort layouts, recursive queries, and query performance optimizations on modern column-oriented engines (Snowflake, BigQuery, DuckDB) and transactional engines (PostgreSQL).

1. Window Functions & Custom Frames

Window functions allow row transformations relative to partition segments without grouping rows. Understanding the exact sliding window frame controls data volume and calculations.

-- Calculate 7-day moving average of revenues
SELECT
  order_date,
  daily_revenue,
  AVG(daily_revenue) OVER(
    ORDER BY order_date
    ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
  ) AS rolling_7day_avg
FROM daily_sales;

-- Cumulative running sum over partitions
SELECT
  user_id,
  event_time,
  SUM(points) OVER(
    PARTITION BY user_id
    ORDER BY event_time
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS lifetime_points
FROM user_events;
Optimization Tip: Prefer ROWS over RANGE. ROWS checks physical row numbers which is significantly faster and uses less memory. RANGE calculates values relative to logical values and requires extra processing passes.

2. Filtering Window Results using QUALIFY

In modern analytical warehouses (Snowflake, BigQuery, DuckDB), you can avoid wrapping queries in outer subqueries to filter window results by utilizing the QUALIFY clause directly.

-- Get only the most recent purchase for each customer (Instant filtering)
SELECT
  customer_id,
  order_id,
  order_date,
  total_amount
FROM orders
QUALIFY ROW_NUMBER() OVER(
  PARTITION BY customer_id
  ORDER BY order_date DESC
) = 1;

3. Hierarchical Graph Parsing via Recursive CTEs

Use recursive Common Table Expressions to parse trees, directory hierarchies, org charts, or multi-hop clickstreams.

WITH RECURSIVE OrgChart AS (
  -- Anchor member: Start with top-level managers
  SELECT employee_id, manager_id, employee_name, 1 AS org_depth
  FROM employees
  WHERE manager_id IS NULL

  UNION ALL

  -- Recursive member: Link employees to managers
  SELECT e.employee_id, e.manager_id, e.employee_name, o.org_depth + 1
  FROM employees e
  JOIN OrgChart o ON e.manager_id = o.employee_id
)
SELECT * FROM OrgChart ORDER BY org_depth, employee_name;

4. Analytical Cohort Transformations

Designing user cohorts by signup date and calculating transaction offsets is the foundation of user retention calculations.

WITH UserSignups AS (
  SELECT user_id, DATE_TRUNC('month', created_at) AS cohort_month
  FROM users
),
UserRevenues AS (
  SELECT
    user_id,
    DATE_TRUNC('month', order_date) AS activity_month,
    SUM(amount) AS monthly_revenue
  FROM orders
  GROUP BY 1, 2
)
SELECT
  s.cohort_month,
  DATEDIFF('month', s.cohort_month, r.activity_month) AS month_offset,
  COUNT(DISTINCT s.user_id) AS active_users,
  SUM(r.monthly_revenue) AS cohort_revenue
FROM UserSignups s
JOIN UserRevenues r ON s.user_id = r.user_id
GROUP BY 1, 2
ORDER BY 1, 2;
