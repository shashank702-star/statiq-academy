CLI & Git Version Control

Developing data pipelines requires command line navigation and code version management. This guide covers standard bash/shell commands and Git workflow loops.

1. Shell Navigation (Command Line)

The command line allows you to run scripts, manage packages, and explore system folders.

Command Description Example
pwd Print Working Directory: Outputs your current active folder path. pwd
ls or dir List: Displays files and folders in your current directory. ls -la
cd Change Directory: Navigate to a different folder path. cd data-academy/content
mkdir Make Directory: Creates a new empty folder. mkdir models
rm Remove: Deletes files or directories. Use with caution! rm temp_data.csv

2. Basic Git Workflow Loop

Git is a version control system tracking file changes. Workflows follow a simple Stage → Commit → Push pipeline.

The Git Work Loop

  1. Modify your project source code files locally.
  2. Stage files: git add file.py (prepares them for tracking).
  3. Commit snapshot: git commit -m "commit message" (saves locally).
  4. Upload: git push origin main (pushes commit to remote cloud hosting, e.g., GitHub).

3. Standard Git Commands

Use these commands to check status and sync repositories.

# Initialize git in a new folder
git init

# Clone a remote repository locally
git clone https://github.com/username/repo.git

# Check which files have been modified
git status

# Stage all modified and untracked files
git add .

# Save local changes with a message
git commit -m "Feature: add neural networks guide"

# Check history of commits
git log -n 5
