PowerBI DAX & Semantic Modeling

A reference guide on DAX functions, Calculated Columns vs. Measures, filter contexts, and Time Intelligence expressions.

📈 Semantic Data Modeling

Data Analysis Expressions (DAX) is the formula language used throughout PowerBI to build contextual columns and metrics inside your star schema.

1. Calculated Columns vs. Measures

Understanding when to use columns versus measures is critical for RAM optimization:

  • Calculated Column (Row Context): Calculated row-by-row during data load. Consumes RAM. Used for slicing/filtering dimensions.
  • Measure (Filter Context): Computed dynamically on-the-fly when added to a visual. Optimized, ignores row context by default. Used for quantitative aggregation metrics.

2. The CALCULATE Function (Filter Modification)

CALCULATE is the most powerful function in DAX, allowing you to modify active filter contexts.

-- Calculate sales specifically for the Technology category
TechSales = CALCULATE(
    SUM(Sales[sales_usd]),
    Product[category] = "Technology"
)

-- Calculate sales ignoring all filters currently applied to the visual
AllSales = CALCULATE(
    SUM(Sales[sales_usd]),
    ALL(Sales)
)

3. Related & Relationships (RELATED / RELATEDTABLE)

Pull columns from lookup dimension tables into fact tables using active star-schema relationships.

-- Calculated Column in Fact table pulling segment from Customer dimension
CustomerSegmentColumn = RELATED(Customer[customer_segment])

4. Time Intelligence Expressions

DAX contains specialized functions to compare year-over-year and month-over-month sales trends seamlessly.

-- Year-to-Date (YTD) Revenue
YTD_Revenue = TOTALYTD(SUM(Sales[sales_usd]), DateTable[Date])

-- Revenue from the same period last year
PriorYearRevenue = CALCULATE(
    SUM(Sales[sales_usd]),
    SAMEPERIODLASTYEAR(DateTable[Date])
)
