Prompt Engineering & LLM APIs

A reference guide on zero-shot/few-shot prompting, system instructions, structured output parsing, and calling LLM APIs in Python.

🤖 Language Models as Engines

Prompt engineering is the practice of designing inputs to elicit specific, structured responses from LLMs. Effective prompts control tone, output structure, and logical accuracy.

1. Key Prompting Design Patterns

  • Zero-Shot: Ask the model to perform a task without giving examples.
  • Few-Shot (In-Context Learning): Provide 2-3 examples of desired inputs and outputs before your actual prompt to enforce format consistency.
  • Chain-of-Thought (CoT): Direct the model to "explain your reasoning step-by-step" before returning the final answer, reducing logical hallucination.

2. Enforcing Structured JSON Output

In production applications, prompts should enforce structured schemas (like JSON) for downstream programmatic execution.

# Prompt Structure Example:
Analyze the sentiment of the customer feedback below. 
You must respond in valid JSON format matching this schema:
{
  "sentiment": "positive" | "neutral" | "negative",
  "confidence_score": float (0.0 to 1.0),
  "primary_topic": string
}
Feedback: "The database query latency is down by 40% after implementing indexes. Outstanding job!"

3. Calling LLM APIs in Python

Connect and interact with large language models programmatically.

from google import genai
from google.genai import types

client = genai.Client()

response = client.models.generate_content(
    model='gemini-2.5-flash',
    contents='Explain Bayes Theorem simply in two sentences.',
    config=types.GenerateContentConfig(
        system_instruction="You are a precise data science coach.",
        temperature=0.2, # low temperature for deterministic answers
    ),
)

print(response.text)
