📊 The Statistical Computing Standard

R is a language and environment designed specifically for statistical computing, data analysis, and graphics. Unlike general-purpose languages, R treats vectors and statistical distributions as first-class citizens, making it the preferred choice for academic research, clinical trials, and deep statistical modeling.

1. R Core Syntax & Data Structures

R uses the assignment operator <- to assign values to variables. Core data structures include:

  • Vectors: The most basic data structure. Must contain elements of the same data type. Vectors are 1-indexed. 
    # Declaring numeric and character vectors
ages <- c(25, 30, 35, 40)
names <- c("Alice", "Bob", "Charlie", "David")

# Vectorized operation (adds 5 to all elements automatically)
older_ages <- ages + 5
  • Factors: Used to store categorical data with predefined levels (e.g. "Low", "Medium", "High").
  • Lists: Heterogeneous collections containing elements of different types (e.g. vectors, matrices, or other lists). 
    # Lists use double brackets [[]] for extracting elements
my_list <- list(title = "Metrics", values = c(8.5, 9.2, 7.8))
my_title <- my_list[[1]]      # Extracts "Metrics"
my_vals <- my_list$values     # Extracts vector
  • Data Frames: Tabular data structures where each column acts as a vector. This is R's native database structure. 
    # Creating a data frame
df <- data.frame(Name = names, Age = ages)
# Slicing: df[rows, columns]
first_row <- df[1, ]
age_col <- df[, "Age"]

2. Data Wrangling with Tidyverse & dplyr

The Tidyverse is a curated collection of R packages designed for data science. The dplyr package provides a grammar of data manipulation. R uses the pipe operator %>% (magrittr) or the native pipe |> (R 4.1+) to chain operations sequentially.

library(tidyverse)

# Cleaning and wrangling a dataset using dplyr pipes
cleaned_data <- raw_data %>%
  # 1. Filter rows (WHERE)
  filter(sales_usd > 1000, region == "North") %>%
  
  # 2. Select specific columns (SELECT)
  select(customer_id, product_name, sales_usd, quantity) %>%
  
  # 3. Create or modify columns (MUTATE)
  mutate(price_per_unit = sales_usd / quantity) %>%
  
  # 4. Group & Summarise (GROUP BY & Aggregations)
  group_by(product_name) %>%
  summarise(
    total_sales = sum(sales_usd),
    avg_price = mean(price_per_unit),
    transaction_count = n()
  ) %>%
  
  # 5. Sort results (ORDER BY)
  arrange(desc(total_sales))

3. Data Visualization with ggplot2

The ggplot2 package implements the Grammar of Graphics, building plots in layers by mapping data variables to visual aesthetic attributes (x, y, color, size).

# Creating a scatter plot with a linear regression trend line
ggplot(data = df, mapping = aes(x = Age, y = sales_usd, color = Region)) +
  geom_point(size = 3, alpha = 0.7) +
  geom_smooth(method = "lm", se = TRUE, color = "white") +
  facet_wrap(~ ProductType) +
  labs(
    title = "Customer Spend vs. Age",
    x = "Customer Age (Years)",
    y = "Spend (USD)"
  ) +
  theme_minimal()

4. Statistical Modeling & Regression

R excels at running linear models and extraction of statistical summaries. The lm() function fits linear regression models, modeling dependencies using formula notation y ~ x.

# Fitting a linear regression model: Spend predicted by Age and Income
model <- lm(spend_usd ~ age + annual_income, data = df)

# Output summary containing Coefficients, R-squared, p-values, and F-statistic
summary(model)

# Running an Analysis of Variance (ANOVA)
anova(model)

# Hypothesis testing: Independent two-sample t-test
t.test(spend_usd ~ group, data = df, alternative = "two.sided")

5. Reproducible Research (R Markdown & Quarto)

R Markdown and Quarto integrate code execution, statistical outputs, plots, and markdown narrative into single document templates. This ensures analysis notebooks can be compiled dynamically into HTML, PDF, or Word files, making statistical reporting perfectly reproducible.

📋 R Programming Quick Cheatsheet

Core Operators

  • <- : Variable assignment
  • %>% or |> : Pipe output to next function
  • $ : Extract column or list element
  • %in% : Value matching in vector

Core Functions

  • c(...) : Combine/create vector
  • summary(x) : Get statistical summary
  • str(x) : Show structural layout of object
  • head(x, n) : View first n rows
