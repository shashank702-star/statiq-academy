LLM Retrieval-Augmented Generation (RAG) Blueprint

Retrieval-Augmented Generation (RAG) solves LLM hallucination and updates factual knowledge by searching and stuffing relevant document context into the model's prompt context window before execution. This walkthrough outlines the architecture of a production-level RAG system.

🧩 RAG Pipeline Architecture

A production-ready RAG pipeline follows four sequential stages: chunking/ingestion, vector indexing, query retrieval, and LLM prompt generation.

1 Ingestion & Chunking

Raw text documents (like PDFs, manuals, or database logs) are split into semantic chunks. Sliding window chunking ensures that paragraphs are not cut in half, preserving context bounds.

# In Python using LangChain recursive chunkers
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,      # chunk length constraint
    chunk_overlap=200,    # sliding boundary overlay
    length_function=len,
    is_separator_regex=False,
)

chunks = text_splitter.split_text(raw_document_text)

2 Generating Dense Embeddings

Convert raw text chunks into mathematical vectors representing semantic meaning using modern embeddings APIs.

# Using OpenAI text-embedding-3-small API (outputs 1536 dimensions)
import openai

response = openai.Embedding.create(
    model="text-embedding-3-small",
    input="Sample text chunk to embed in database index"
)
embedding_vector = response['data'][0]['embedding']

3 Vector Database Indexing (pgvector)

Store vector outputs in a vector-capable database. For enterprise PostgreSQL setups, use the pgvector extension configured with an HNSW (Hierarchical Navigable Small World) index for fast approximate nearest neighbor searches.

-- Enable extension and create document table
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE document_chunks (
    id SERIAL PRIMARY KEY,
    content TEXT,
    embedding vector(1536) -- matching dimension size
);

-- Create highly optimized HNSW index for Cosine Distance search
CREATE INDEX ON document_chunks USING hnsw (embedding vector_cosine_ops);

4 Similarity Retrieval

When a user asks a query, embed their query and run a cosine similarity query on the index to pull the top 3 matching chunks.

-- Retrieve top matching segments using Cosine operator (<=>)
SELECT content, 1 - (embedding <=> '[0.015, -0.023, ...]') AS similarity
FROM document_chunks
ORDER BY embedding <=> '[0.015, -0.023, ...]'
LIMIT 3;

5 Prompt Context Stuffing & Execution

Stuff the retrieved texts directly into the LLM system prompt as reference material, ensuring the response is anchored to facts.

system_prompt = f"""
You are an expert system. Answer the user's question using ONLY the context provided below.
If the context does not contain the answer, reply that you do not know.

DOCUMENT CONTEXT REFERENCE:
1. {retrieved_chunks[0]}
2. {retrieved_chunks[1]}
3. {retrieved_chunks[2]}
"""
