Tableau Dashboard Design & Calculations

A masterclass reference covering calculated fields, Level of Detail (LOD) calculations, Parameter Actions, and dashboard visual layouts.

📈 Visual Storytelling with Data

A great dashboard must be fast, readable, and action-oriented. Learn how to optimize query speeds and design UI layouts that executives can read in under 5 seconds.

1. Level of Detail (LOD) Calculations

LOD expressions allow you to compute values at the source level and the visualization level simultaneously, ignoring filters when necessary.

# FIX: Calculate Lifetime Value per Customer ignoring viz details
{ FIXED [Customer ID] : SUM([Order Revenue]) }

# EXCLUDE: Calculate Regional Sales ignoring specific Store details
{ EXCLUDE [Store Name] : SUM([Sales]) }

2. Parameter Actions for Interactive UX

Parameters make charts dynamic. Set up **Parameter Actions** to let users hover or click on a chart mark to change the active metric displayed on another sheet.

# CASE calculation to swap metrics dynamically based on Parameter selection
CASE [Select Metric Parameter]
  WHEN "Sales" THEN SUM([Sales])
  WHEN "Profit" THEN SUM([Profit])
  WHEN "Quantity" THEN SUM([Quantity])
END

3. Table Calculations & Indexing

Compute running totals, percentage differences, or ranks directly on the aggregated view without fetching raw database rows.

# Calculate Percent of Total Sales for current row
SUM([Sales]) / TOTAL(SUM([Sales]))

# Rank customers based on Revenue
RANK(SUM([Sales]))

4. Dashboard UI Layout Best Practices

🎨 Visual Hierarchy Checklist

  • F-Layout reading path: Place high-level KPI cards at the top-left, and detailed tables at the bottom-right.
  • Limit Color Usage: Stick to a cohesive 3-color palette (e.g. Slate background, Blue highlights, and Red/Green indicator highlights). Avoid multi-colored rainbows.
  • White Space: Leave at least 10px-15px padding between dashboard objects to let layout elements breathe.
