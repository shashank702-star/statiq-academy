Tableau Dashboard Design & Calculations

A comprehensive masterclass reference covering Tableau Desktop architecture, logical relationships, Level of Detail (LOD) calculations, Table Calculations, and interactive dashboard design systems.

1. Tableau Architecture & Data Modeling

Modern Tableau uses a dual-layer data modeling system to query datasets efficiently:

  • Logical Layer (Relationships): The default interface where you link tables using "noodles." Relationships defer joining tables until compile-time, dynamically selecting the correct SQL JOIN based on the fields used in the active visualization. This prevents record duplication (fan-out) hazards.
  • Physical Layer (Joins & Unions): Double-clicking a logical table opens the physical layer, where you can define hardcoded Inner, Left, Right, or Full Outer joins. Use this when you must merge tables at the physical source level.
  • Live vs. Extract Connection: Live queries databases directly in real-time. Extracts pull data into local memory using Tableau's high-speed columnar data engine, **Hyper**, which optimizes memory caching for millions of rows.

2. Level of Detail (LOD) Expressions & Order of Operations

Level of Detail expressions allow you to compute aggregations at specific dimensional granularities, independent of the fields dragged onto the visualization canvas.

# FIXED LOD: Aggregates value strictly at specified dimension, bypassing view layout filters
{ FIXED [Region] : SUM([Sales]) }

# INCLUDE LOD: Computes values at a lower (more detailed) granularity than what is displayed
{ INCLUDE [Customer Name] : AVG([Order Profit]) }

# EXCLUDE LOD: Computes values at a higher (less detailed) granularity than the active sheet
{ EXCLUDE [Product Category] : SUM([Sales]) }

⚠️ Tableau Order of Operations

Tableau evaluates query filters in a strict sequential order. Knowing this is crucial for making FIXED LODs respond to filters:

  1. Extract Filters
  2. Data Source Filters
  3. Context Filters (FIXED LODs evaluate *after* Context Filters)
  4. FIXED LOD Calculations
  5. Dimension Filters (FIXED LODs evaluate *before* Dimension Filters!)
  6. INCLUDE/EXCLUDE LOD Calculations
  7. Measure Filters
  8. Table Calculations (e.g. running totals, ranks)

Tip: If a dimension filter is not updating a FIXED LOD, right-click the filter and select Add to Context to force it to evaluate before the LOD calculation.

3. Table Calculations

Table calculations compute values on the local aggregated database returned to the visualization viewport, rather than querying raw rows from the server. They are defined by "Compute Using" settings (Addressing and Partitioning):

# Percentage of Total sales in view
SUM([Sales]) / TOTAL(SUM([Sales]))

# Running Sum calculation (cumulative total)
RUNNING_SUM(SUM([Sales]))

# Window average of sales across partition elements
WINDOW_AVG(SUM([Sales]), -3, 0) # Average of last 3 rows + current row

4. Interactive UX: Parameter & Set Actions

To create dynamic dashboards that feel alive, leverage actions to feed selection details back to variables:

  • Parameter Actions: Allow user mouse selections on one sheet to overwrite a parameter value, dynamically updating other sheets using that parameter (e.g. swapping between Sales/Profit views).
  • Set Actions: Dynamically add or remove selected marks from a Set, enabling asymmetric drill-downs or highlighting related segments.
  • Dynamic Zone Visibility: Hide or show containers dynamically based on boolean parameter values (ideal for toggling between chart types or filters panels).

5. Dashboard Design & Styling Systems

  • Visual Hierarchy (Z-Layout): Position primary KPIs at the top left, main interactive charts in the center, and granular tables at the bottom.
  • Layout Containers: Always build dashboards using nested Horizontal and Vertical containers rather than floating objects to maintain layout responsiveness.
  • Decluttering: Remove default gridlines, borders, and legends when possible. Place direct labels on charts to decrease cognitive strain.
