Data Pipelines & Orchestration (Airflow)

A reference guide on declaring Directed Acyclic Graphs (DAGs), managing task dependencies, scheduling runs, and writing tasks using Airflow operators.

⛓️ Pipeline Orchestration

Apache Airflow is a platform to programmatically author, schedule, and monitor data workflows. Workflows are represented as Directed Acyclic Graphs (DAGs) in Python code.

1. Anatomy of a DAG

Every Airflow pipeline consists of a DAG object that encapsulates task dependencies and execution configurations.

from datetime import datetime, timedelta
from airflow import DAG
from airflow.operators.python import PythonOperator

default_args = {
    'owner': 'statiq_ops',
    'depends_on_past': False,
    'start_date': datetime(2026, 1, 1),
    'retries': 2,
    'retry_delay': timedelta(minutes=5),
}

# Declare DAG connection & schedule
with DAG(
    dag_id='statiq_retail_pipeline',
    default_args=default_args,
    description='ETL pipeline for retail store transactions',
    schedule_interval='@daily',
    catchup=False,
) as dag:
    # Operators go here
    pass

2. Python & SQL Operators

Operators represent a single task within the pipeline. Airflow provides operators for shell scripts, SQL statements, and Python execution.

def extract_transactions():
    print("Fetching CSV ledger data...")

def transform_metrics():
    print("Normalizing currency rates...")

# Define python tasks
task_extract = PythonOperator(
    task_id='extract_raw_data',
    python_callable=extract_transactions,
)

task_transform = PythonOperator(
    task_id='transform_ledger_data',
    python_callable=transform_metrics,
)

3. Defining Task Dependencies

Declare task execution sequences using bitwise operators. Tasks can run sequentially or in parallel groups.

# Run task_extract FIRST, then run task_transform SECOND
task_extract >> task_transform
