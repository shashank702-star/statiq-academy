Feature Engineering & Data Preprocessing

A reference guide on preparing numeric and categorical variables, scaling features, transforming skewed distributions, and extracting indicators.

🛠️ Garbage In, Garbage Out

Feature Engineering is the process of using domain knowledge to extract features from raw data that help machine learning algorithms learn faster and predict more accurately.

1. Categorical Encoding (One-Hot vs. Ordinal)

Algorithms require numeric inputs. Encode categories based on relationship structures:

  • One-Hot Encoding: Converts nominal columns into separate binary flags (0 or 1). Best when categories have no ranking.
  • Ordinal Encoding: Converts ordinal columns into integers representing rank orders.
import pandas as pd

df = pd.DataFrame({'Device': ['Mobile', 'Desktop', 'Mobile', 'Tablet']})

# Apply One-Hot Encoding
encoded_df = pd.get_dummies(df, columns=['Device'], drop_first=True)
print(encoded_df)

2. Handling Skewed Columns (Log Transform)

Skewed features (like income, transaction sizes, or house prices) violate regression normality assumptions. Compress long tails using log transformations.

import numpy as np

# Apply Log Transform (plus 1 to avoid log(0) errors)
df['log_revenue'] = np.log1p(df['raw_revenue'])

3. Feature Scaling (Standardization vs. Normalization)

Distance-based algorithms (like KNN, SVM, K-Means) are sensitive to column scales. Scale columns before fitting models:

  • Standardization (Z-score Scaling): Rescales columns to have a mean of 0 and standard deviation of 1. Best for normal-like distributions.
  • Normalization (Min-Max Scaling): Rescales columns between 0 and 1. Best when columns have bounded intervals.
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
df_scaled = scaler.fit_transform(df[['age', 'income']])
