Data Analysis Lifecycle & Methods

Data analysis is a structured methodology for collecting, transforming, modeling, and explaining variables to uncover actionable patterns that inform organizational choice.

This presentation deck outlines the core workflow phases and basic methods used by professional analysts daily.

1 / 6

The 6 Lifecycle Phases

A standardized analytical workflow consists of six distinct phases:

  • Ask: Formulate specific, measurable business questions.
  • Prepare: Identify and extract data (databases, CSVs, API endpoints).
  • Process: Clean files, handle missing rows, filter anomalies.
  • Analyze: Perform calculations, group matrices, and run queries.
  • Share: Design dashboards and present summaries.
  • Act: Execute choices based on the statistical insights.

Descriptive Analytics (What Happened?)

Descriptive metrics explain historical outcomes using simple summary aggregates:

  • Mean: The average value of a distribution.
  • Median: The middle value (splits the ordered dataset 50/50). Resistant to outliers.
  • Standard Deviation: Measures the dispersion of data points around the mean.
3 / 6

Diagnostic Analytics (Why Did It Happen?)

Diagnostic analytics digs deeper to isolate factors causing specific trends:

  • Cohort Analysis: Grouping users by start date to compare monthly retention.
  • Correlation Studies: Finding dependencies between metrics (e.g. support ticket frequency vs churn).
  • Drill-downs: Dissecting macro indicators by dimensions like geography, age, or segment.

Python Data Wrangling Guide

Basic data transformations using Python's Pandas package:

import pandas as pd

# Load retail orders data
df = pd.read_csv("content/store-sales-data.csv")

# Group by category & sum profit
category_profit = df.groupby("category")["profit"].sum().reset_index()
print(category_profit)

Clean Data Checklist

Before presenting dashboards or fitting predictive models, verify the dataset satisfies these data quality benchmarks:

  • Completeness: Zero unintended null values in critical ID/metric fields.
  • Uniqueness: Primary keys are validated and contain zero duplicates.
  • Consistency: Categorical entries are normalized (no duplicate categories due to typos).
  • Accuracy: Numeric metrics fall within valid business bounds (e.g., negative checkout prices).
