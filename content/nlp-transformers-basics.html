NLP & Transformer Architectures

Natural Language Processing (NLP) bridges human language data and computers. Modern language AI models rely on numerical token mappings and the Transformer self-attention architecture.

1. Text Preprocessing: Tokenization & Embeddings

Computers cannot process raw text strings. Text must be cleaned, split into tokens, and represented as vector arrays.

  • Tokenization: Splitting text strings into words, subwords, or characters.
  • Bag of Words / TF-IDF: Representing documents by word frequencies adjusted by their dataset rarity.
  • Word Embeddings: Representing words as dense vectors (e.g. 300 dimensions) where geometrically close vectors share semantic meanings (e.g. "King" - "Man" + "Woman" ≈ "Queen").

2. Vector Mappings in Python

Using TF-IDF features to represent documents for text classification tasks:

from sklearn.feature_extraction.text import TfidfVectorizer

# Sample Documents
documents = [
    "Machine learning parses dataset metrics.",
    "SQL database structures table rows.",
    "Deep learning models train neural weights."
]

# Calculate TF-IDF
vectorizer = TfidfVectorizer()
tfidf_matrix = vectorizer.fit_transform(documents)

# View vocabulary mappings
print(vectorizer.get_feature_names_out())
print(tfidf_matrix.toarray())

3. The Self-Attention Mechanism

Traditional RNN models read text sequentially, forgetting early words in long paragraphs. The **Self-Attention** mechanism in Transformers processes all words simultaneously, calculating context weights between all words in a sentence.

The Attention Formula

Self-attention maps input queries ($Q$), keys ($K$), and values ($V$) to compute dynamic context scores:

Attention(Q, K, V) = Softmax( (Q * K^T) / √d_k ) * V

This allows the word "bank" to dynamically adapt its embedding representation based on surrounding context keys like "river bank" vs. "money bank".

4. Transformer Architecture (Encoder-Decoder)

The Transformer consists of two main stack networks:

  • Encoder: Processes input text to build semantic representation vectors (e.g. BERT models). Excellent for classification, entity recognition, and sentiment checks.
  • Decoder: Takes encoder vectors and autoregressively generates output text (e.g. GPT models, Gemini). Excellent for translations, text generations, and conversation completions.
