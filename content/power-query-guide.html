Power Query & M-Language Transformations

Power Query is the data preparation and ETL (Extract, Transform, Load) engine for Power BI, Excel, and other Microsoft services. It features a graphical user interface for transforming data, backed by the powerful, functional M-formula language.

1. Power Query ETL & Query Folding

Power Query acts as the gateway to your BI models. One of its most critical performance features is Query Folding.

⚡ What is Query Folding?

Query Folding is the ability of Power Query to compile the steps you define in the graphical editor into a single, native database query (e.g. SQL SELECT with joins, filters, and groups) and execute it directly on the source server. This minimizes data transfer and uses the database engine's optimized speed.

Actions that fold: Filtering rows (WHERE), selecting columns (SELECT), joining tables (JOIN), grouping rows (GROUP BY).

Actions that break folding: Changing data types, merging columns from different sources, adding index columns, or writing custom M code that cannot be translated to SQL. Ensure non-folding steps are placed at the very end of your query.

2. Data Transformations in Power Query

Power Query provides GUI-driven steps that are converted to M code under the hood. Key transformations include:

  • Merge Queries: Joins two tables horizontally using key columns (equivalent to SQL JOINs). You can select Inner, Left Outer, Right Outer, Full Outer, or Anti-joins.
  • Append Queries: Stacks two or more tables vertically (equivalent to SQL UNION ALL). Columns must have matching names and data types.
  • Unpivot Columns: Translates crosstab/matrix data formats into flat tabular records. This is vital for BI dashboards, transforming columns like "Jan Sales", "Feb Sales", "Mar Sales" into a single "Month" column and "Sales" value column.
  • Group By: Aggregates row values (Sum, Average, Min, Max, Count) based on categorical grouping columns.

3. The M-Formula Language Basics

Every step in Power Query is written in **M** (Data Mashup Formula Language). M is a functional, case-sensitive language where variables are immutable.

# The let...in block structure of M
let
    // 1. Load CSV data source
    Source = Csv.Document(File.Contents("C:\Ledger.csv"), [Delimiter=",", Columns=4, Encoding=1252, QuoteStyle=QuoteStyle.None]),
    
    // 2. Promote headers
    PromotedHeaders = Table.PromoteHeaders(Source, [PromoteAllScalars=true]),
    
    // 3. Filter rows (equivalent to SQL WHERE)
    FilteredRows = Table.SelectRows(PromotedHeaders, each ([Status] = "Completed")),
    
    // 4. Cast data types
    TypedColumns = Table.TransformColumnTypes(FilteredRows, {{"Amount", type number}, {"Date", type date}}),
    
    // 5. Add custom column
    AddTax = Table.AddColumn(TypedColumns, "TotalAmount", each [Amount] * 1.18, type number)
in
    AddTax

4. Parameterization & Dynamic Loading

Parameters allow your queries to adapt dynamically without rewriting code. They are highly useful for altering file paths, database server endpoints, or filtering dates during deployment transfers:

# Example: Using a parameter 'ServerName' in a SQL connection
let
    Source = Sql.Database(ServerName, "SalesDB"),
    SalesTable = Source{[Schema="dbo", Item="SalesTransactions"]}[Data]
in
    SalesTable
